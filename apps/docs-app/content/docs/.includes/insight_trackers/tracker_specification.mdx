[InsightTracker schema](https://journium.app/schemas/journium-insight-tracker.schema.json).

## Analysis Types

The `analysis.type` field selects the analysis algorithm. Each type computes different metrics
and injects different template variables into your `llm.promptTemplate`.

| Type | Description |
|---|---|
| [`funnel`](./analysis_types/funnel) | Ordered step completion — find where users drop off |
| [`retention`](./analysis_types/retention) | Cohort return-rate curves (D1 / D7 / D30) |
| [`activation`](./analysis_types/activation) | % of new users who reach the first value moment |
| [`churn`](./analysis_types/churn) | Users who were active but have gone silent |
| [`featureAdoption`](./analysis_types/feature_adoption) | Fraction of active users who have tried a feature |
| [`anomaly`](./analysis_types/anomaly) | Automated spike / dip detection vs. a statistical baseline |
| [`conversion`](./analysis_types/conversion) | Simple numerator / denominator conversion rate |
| [`cohort`](./analysis_types/cohort) | Compare a metric across cohorts by join date or property |
| [`trend`](./analysis_types/trend) | Count events or unique users per time period |
| [`custom`](./analysis_types/custom) | Raw event list analysis — not available in the NL Builder |

See the [Analysis Types overview](./analysis_types) for per-type configuration reference and examples.

### Universal Template Variables

These variables are always available in `promptTemplate`, regardless of analysis type:

| Variable | Description |
|---|---|
| `{{windowPeriod}}` | Human-readable description of the analysis window, e.g. `"last 7 days"` |
| `{{dataAsJson}}` | Full structured result from the analysis query, serialised as JSON |
| `{{executedAt}}` | ISO 8601 timestamp of when this execution started |

Type-specific variables are documented on each analysis type's reference page.

## Full Example

```yaml title=".journium/trackers/checkout-funnel.yml"
apiVersion: journium.app/v0Beta
kind: InsightTracker
metadata:
  name: checkout-funnel
  displayName: Checkout Funnel Analysis
  description: Track drop-off across the 4-step checkout flow
spec:
  type: LLM
  trigger:
    mode: automatic
    schedule: daily
  window:
    period: last_7d
    granularity: day
    timezone: America/New_York
  analysis:
    type: funnel
    entity: person_id
    steps:
      - event: cart_viewed
        label: Viewed Cart
      - event: checkout_started
        label: Started Checkout
      - event: payment_entered
        label: Entered Payment
      - event: order_completed
        label: Completed Order
    conversionWindow: 2h
    breakdowns:
      - property: plan
        maxCardinality: 5
  llm:
    promptTemplate: |
      Analyze this checkout funnel for {{windowPeriod}}.
      {{totalEntrants}} users entered the funnel with an overall conversion of {{overallConversionPct}}%.
      Step breakdown: {{stepsSummary}}
      Worst performing step: {{worstStep}} ({{worstStepCvr}}% CVR, {{worstStepLoss}} users lost).
      Full data: {{dataAsJson}}
      Provide 2-3 concrete, actionable recommendations to improve conversion.
    maxOutputTokens: 500
    guardrails:
      citeSources: true
  privacy:
    excludeProperties:
      - email
      - phone
    hashProperties:
      - userId
```
