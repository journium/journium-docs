## General Questions

<AccordionGroup>
  <Accordion title="What is Journium and how does it work?" icon="info-circle">
    Journium is a conversion optimization platform that helps small businesses improve their website's performance. It works by:
    
    1. **Tracking user behavior** - Monitoring how users interact with your website
    2. **Analyzing patterns** - Using AI to identify optimization opportunities
    3. **Providing recommendations** - Suggesting specific changes to improve conversions
    4. **Testing improvements** - Running A/B tests to validate changes
    
    The platform combines real-time analytics, AI-powered insights, and easy-to-use A/B testing tools.
  </Accordion>
  
  <Accordion title="How is Journium different from Google Analytics?" icon="chart-line">

    |---------|------------------|-----------|

    | **AI Recommendations** | Limited | Core feature |
    | **A/B Testing** | Via Optimize (deprecated) | Built-in |
    | **Small Business Focus** | General purpose | Specifically designed |
    | **Implementation** | Complex setup | Quick integration |
    | **Actionable Insights** | Requires expertise | Automatic recommendations |
    
    Journium complements Google Analytics by providing actionable insights for improving conversion rates.

  <Accordion title="Do I need to install Journium on every page?" icon="globe">
    **Single Installation:**
    Install Journium once in your site's header or main layout template, and it will automatically track across all pages.
    
    **CDN Installation:**
    ```html
    <!-- Add to your main template/layout -->
    <head>
      <script src="https://cdn.journium.com/sdk/v1/journium.min.js"></script>
      <script>
        Journium.init({
          projectId: 'your-project-id',
          apiKey: 'your-api-key',
          environment: 'production'
        });
      </script>
    </head>
    ```
    
    **React/Next.js:**
    Add the provider at your app root and it covers all child components.
    
    **SPA Considerations:**
    For single-page applications, ensure the SDK is initialized before route changes occur.
  </Accordion>

  <Accordion title="How accurate are the AI recommendations?" icon="brain">
    **Accuracy Factors:**
    
    **Data Quality (85-95% accuracy with sufficient data):**
    - Minimum 1,000 monthly visitors for reliable recommendations
    - At least 100 conversions per month for conversion-focused suggestions
    - 2+ weeks of data collection for pattern recognition
    
    **Recommendation Types:**
    - **Technical improvements**: 90-95% accuracy (measurable metrics)
    - **UX enhancements**: 80-90% accuracy (some subjectivity)
    - **Content/copy changes**: 70-85% accuracy (audience-dependent)
    
    **Confidence Scoring:**
    - Each recommendation includes a confidence score (0-100%)
    - Higher confidence = more data supporting the recommendation
    - Lower confidence = suggestion based on industry patterns
    
    **Validation Process:**
    - All recommendations should be A/B tested before full implementation
    - Journium tracks implementation results to improve future accuracy
    - Success rates vary by industry and audience
  </Accordion>
  
  <Accordion title="Can I run multiple A/B tests simultaneously?" icon="flask">
    **Yes, but with important considerations:**
    
    **Best Practices:**
    - Test different page elements (not overlapping)
    - Avoid testing competing hypotheses
    - Ensure sufficient traffic for each test
    - Monitor for interaction effects
    
    **Safe Combinations:**
    ```javascript
    // ✅ Good: Different pages/elements
    Test 1: Homepage headline
    Test 2: Product page CTA button
    Test 3: Checkout form fields
    
    // ❌ Avoid: Same element or competing changes
    Test 1: Homepage CTA button color
    Test 2: Homepage CTA button text
    Test 3: Homepage CTA button placement
    ```
    
    **Traffic Requirements:**
    - Each test needs sufficient traffic for statistical significance
    - Running 3 tests simultaneously requires ~3x more traffic
    - Consider sequential testing for lower-traffic sites
    
    **Journium automatically:**
    - Prevents overlapping tests on same elements
    - Calculates required traffic for multiple tests
    - Suggests optimal testing schedule
  </Accordion>
  
  <Accordion title="How long should I run an A/B test?" icon="stopwatch">
    **Minimum Requirements:**
    - **Duration**: At least 1 week (preferably 2 weeks)
    - **Sample Size**: Minimum 1,000 visitors per variant
    - **Conversions**: At least 100 conversions per variant
    - **Statistical Significance**: 95% confidence level
    
    **Factors Affecting Duration:**
    
    **Traffic Volume:**
    - High traffic (>10k/day): 1-2 weeks
    - Medium traffic (1-10k/day): 2-4 weeks  
    - Low traffic (<1k/day): 4-8 weeks
    
    **Conversion Rate:**
    - Higher conversion rates = faster results
    - Lower conversion rates = longer testing needed
    
    **Effect Size:**
    - Large improvements (>20%) = detected faster
    - Small improvements (<10%) = longer testing needed
    
    **Business Cycles:**
    - Include full business cycles (weekly patterns)
    - Avoid holiday periods or unusual events
    - Account for seasonal variations
    
    **Journium provides:**
    - Automatic duration estimates
    - Progress tracking
    - Early stopping recommendations
    - Statistical significance monitoring
  </Accordion>

  <Accordion title="Can users opt out of tracking?" icon="user-slash">
    **Yes, multiple opt-out mechanisms are available:**
    
    **Browser-Based Opt-Out:**
    ```javascript
    // Respect Do Not Track header
    Journium.init({
      // ... config
      config: {
        respectDoNotTrack: true
      }
    });
    
    // Check user preference
    if (navigator.doNotTrack === '1') {
      // Don't initialize tracking
    }
    ```
    
    **Custom Opt-Out:**
    ```javascript
    // Provide opt-out mechanism
    function handleUserOptOut() {
      Journium.optOut();
      localStorage.setItem('journium_opted_out', 'true');
    }
    
    // Check opt-out status before initializing
    if (!localStorage.getItem('journium_opted_out')) {
      Journium.init(config);
    }
    ```
    
    **Consent Management:**
    ```javascript
    // Wait for user consent
    function initWithConsent() {
      showConsentBanner().then(consent => {
        if (consent.analytics) {
          Journium.init(config);
        }
      });
    }
    ```
    
    **Per-Feature Opt-Out:**
    - Users can opt out of specific tracking features
    - Granular consent for different data types
    - Easy opt-out links in privacy policy

    **Best Practices:**
    ```javascript

    <script async src="https://cdn.journium.com/sdk/v1/journium.min.js"></script>
    
    // Optimized configuration
    Journium.init({
      // ... config
      config: {
        batchSize: 50,           // Larger batches
        flushInterval: 10000,    // Less frequent sends
        sampleRate: 0.1          // Sample 10% for high traffic
      }
    });
    ```
    
    **Monitoring:**
    - Built-in performance monitoring
    - Core Web Vitals tracking

    ```javascript
    // Check browser support
    const isSupported = () => {
      return !!(
        window.JSON &&
        (window.fetch || window.XMLHttpRequest) &&
        window.addEventListener
      );
    };
    
    if (isSupported()) {
      Journium.init(config);
    } else {
      // Fallback tracking or disable
      console.warn('Browser not fully supported');
    }
    ```
    
    **Polyfills Available:**
    - Fetch API for older browsers
    - Promise support for IE11
    - Object.assign for legacy support

  <Accordion title="Can I use Journium with Content Security Policy (CSP)?" icon="shield-alt">
    **CSP Configuration:**
    
    **Required Directives:**
    ```html
    <!-- Minimum CSP for Journium -->
    <meta http-equiv="Content-Security-Policy" 
          content="
            script-src 'self' https://cdn.journium.com;
            connect-src 'self' https://api.journium.com https://events.journium.com;
            img-src 'self' https://cdn.journium.com data:;
          ">
    ```
    
    **Full CSP Example:**
    ```html
    <meta http-equiv="Content-Security-Policy" 
          content="
            default-src 'self';
            script-src 'self' https://cdn.journium.com;
            connect-src 'self' https://api.journium.com https://events.journium.com wss://realtime.journium.com;
            img-src 'self' https://cdn.journium.com data: blob:;
            style-src 'self' 'unsafe-inline';
            font-src 'self' https://fonts.googleapis.com https://fonts.gstatic.com;
          ">
    ```
    
    **Nonce Support:**
    ```html
    <!-- Generate nonce server-side -->
    <script nonce="{{ csp_nonce }}" src="https://cdn.journium.com/sdk/v1/journium.min.js"></script>
    <script nonce="{{ csp_nonce }}">
      Journium.init({ ... });
    </script>
    ```
    
    **Hash Support:**
    ```html
    <!-- For inline scripts -->
    <meta http-equiv="Content-Security-Policy" 
          content="script-src 'self' 'sha256-[hash-of-inline-script]' https://cdn.journium.com">
    ```
  </Accordion>
